# designer

Thin front-door skill for Pocket Product Designer v2.

Use it when a design request could be process, product structure, interaction behavior, visual craft, or critique. It routes to the smallest useful specialist instead of loading every design rule at once.

## Routes

- `design-method-finder` - method, research, workshop, and prototype selection
- `product-designer` - personas, JTBD, CUJ, OOUX, flows, PDRs, UXDRs, wireframes
- `microinteractions-expert` - state transitions, validation, recovery, timing, accessibility behavior
- `impeccable` - optional external polished UI/frontend craft skill
- `critique` - optional external critique skill, with product-designer fallback

## Start prompt

```text
Use designer to route this design request and keep fidelity matched to the next decision: <request>
```
