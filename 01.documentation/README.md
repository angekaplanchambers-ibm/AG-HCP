## 1) How to use this folder

This is a companion packet, not a governing PDR. Use it to align workshop framing and near-term product direction without turning the work into a distant north star.

Use these notes when the conversation needs to answer:

- what Airbnb's feedback means for HCP Terraform direction
- why MCP/API access alone is not enough
- what HCP Terraform can own that generic agents cannot
- how terminal, GitHub, Slack, and HCP Terraform UI handoffs connect
- which vertical UX slice is tight enough to ship and credible enough to matter

## 4) Companion stance

Working phrase:

> HCP Terraform should be the trusted place where infrastructure intent becomes verified change.

Keep the work 70% near-term execution and 30% directional vision.

## 5) Non-goals

- Do not frame the first release as a generic AI chatbot.
- Do not make the first release depend on a full HCP Terraform UI redesign.
- Do not make autonomous production apply the first proof point.
- Do not collapse terminal, Slack, GitHub, and UI into one surface.
- Do not position MCP as the whole product answer.
- Do not turn this into a multi-year north star that loses the near-term build path.
