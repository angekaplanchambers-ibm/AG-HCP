Omar’s core position was that the next six months are not about an ambitious AI north star. They are about basic primitives:

- Can coding engines access TFC/TFE data? Only partially.

Current MCP-style lookup tied mainly to module names is insufficient. The agent needs semantic summaries of module purpose, resource composition, and applicability so that when a user asks for something like “S3 with RDS,” the system can find and recommend the right three or four internal modules.

### The product ladder: level one, level two, level three

Omar articulated a three-level progression:

1. **Level one: primitives / fundamentals / table stakes**
   - agent-capable Terraform CLI
   - TFC/TFE data availability to coding engines
   - usable registry and private module context
   - workspace/resource/relationship/version context
   - Explorer/chat context
   - MCP gaps closed or compensated for

2. **Level two: upgrade and repair agents**
   - Terraform version upgrade agent
   - provider version upgrade agent
   - module version upgrade agent
   - plan-error repair
   - Sentinel policy repair
   - Terraform policy/run-error repair
   - proactive workspace scanning against new provider/Terraform releases
   - safe auto-plan / suggested upgrade flows

3. **Level three: AI SRE / AI DevOps / incident remediation**
   - hook into logs and observability systems such as Datadog
   - listen for events or incidents
   - trigger Terraform workflows for remediation
   - connect Slack/docs/bots and infrastructure change flows
   - likely late-2027 or 2028, not immediate six-month scope

The main decision was not to reject levels two and three, but to keep them downstream of level one. The prerequisite layer is missing.
